🏥 Hospital Management System (HMS)

Technical Documentation & User Manual

Project Overview: The Hospital Management System is a robust Command-Line Interface (CLI) Java application that provides comprehensive hospital administration capabilities. It leverages advanced Object-Oriented Programming (OOP) paradigms and connects to a highly structured MariaDB relational database.

1. System Architecture

The application is built using a multi-tiered architecture to ensure separation of concerns, maintainability, and scalability:

  • Model Layer: Contains the core entities (e.g., Patient, Doctor, Appointment). It utilizes abstract classes and interfaces.
  • DAO Layer (Data Access Object): Handles all direct database interactions using JDBC. It features a GenericDAO<T, ID> interface for standardized CRUD operations.
  • Service Layer: Encapsulates business logic, orchestrating calls between the Menu Layer and the DAO Layer.
  • Menu/UI Layer: Manages the interactive CLI, input validation, and rendering of formatted ASCII tables.

2. Object-Oriented Programming (OOP) Integration

Concept Implementation in Project
Encapsulation All class attributes are private and accessed exclusively through getters and setters, ensuring data integrity.
Inheritance Deep class hierarchy: Person (Abstract) → Staff (Abstract) → Doctor → Surgeon.
Polymorphism The displayInfo() method is overridden across different entity classes. Overloaded methods exist for custom implementations.
Abstraction Utilization of abstract methods like getRole() and interface contracts such as Billable and Treatable.
Composition A Patient has-a MedicalRecord. If the patient is deleted, the record ceases to exist independently.

3. Database Design (MariaDB)

The system utilizes a complex database schema tailored for hospital operations:

  • Stored Procedures: Encapsulate complex transactions like sp_RegisterPatient and sp_BookAppointment to ensure atomicity.
  • Triggers: Automated database actions, such as updating bed statuses automatically upon patient admission or discharge.
  • Views: Pre-compiled queries for complex reporting, such as vw_BedOccupancy and vw_PendingBills.
  • Functions: Scalar and table-valued functions for logic like calculating patient age dynamically from their DOB (fn_CalculateAge).

4. Modules & Functionality

  1. Patient Management: Registration, profile updating, admission, and discharge workflows.
  2. Staff Management: Categorized handling of Doctors, Nurses, and Receptionists.
  3. Appointments: Real-time scheduling matching patients with doctors.
  4. Bed Management: Live tracking of ward occupancy and bed availability.
  5. Pharmacy: Inventory management with automated deduction on prescription and low-stock alerts.
  6. Lab Management: Test requests and result logging.
  7. Billing: Consolidated invoicing covering bed charges, doctor fees, lab tests, and medicines.
  8. Reports: High-level dashboard overviews for administration.

5. Installation & Execution

To run the system, ensure you have the following prerequisites:

  • Java JDK 8 or higher.
  • XAMPP (Apache + MySQL/MariaDB).
  • MySQL Connector/J 9.0.0 (placed in the lib/ directory).

Execute the provided run.bat or run.ps1 script, which configures UTF-8 encoding for enhanced console graphics, compiles the source code, and launches the application.
